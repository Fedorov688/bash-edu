
## 2. Variables and Script Arguments

### 2.1. Assignment and Substitution
```bash
#!/bin/bash
server="web-server01"
echo "Configuring server: $server"
echo "Configuration file: ${server}.conf"  # braces clarify variable boundary
```
Important: no spaces around `=`. **Why? What will happen?**

### 2.2. Command Output into Variables

#### Using backticks syntax
```bash
#!/bin/bash
now=`date +%Y-%m-%d_%H-%M`
echo "Date: $now"
```

#### Using parentheses (round brackets) syntax

```bash
#!/bin/bash
now=$(date +%Y-%m-%d_%H-%M)
free_mem=$(free -h | grep Mem | awk '{print $4}')
echo "Date: $now, Free memory: $free_mem"
```

The backtick syntax  \`command\`  also works, but `$(...)` is preferred—it nests more easily.

### 2.3. Command-Line Arguments
Special variables when running `./script.sh param1 param2`:
- `$0` — script name.
- `$1, $2, ...` — positional arguments.
- `$#` — number of arguments.
- `$@` — all arguments (each treated as a separate word). Array.
- `$*` — all arguments as a single string.
- `$?` — exit code of the last command.

**Example** (`args.sh`):

```bash
#!/bin/bash
echo "Script name: $0"
echo "First argument: $1"
echo "All arguments: $@"
echo "-----------------"
for i in "$@"; do ((c1++)); done
echo "Count iterate args = ${c1}"
echo "All arguments (\$@): $@"
echo "-----------------"
for i in "$*"; do ((c2++)); done
echo "Count iterate args = ${c2}"
echo "All arguments (\$*): $*"
echo "-----------------"
echo "Number of arguments: $#"
```

Run: `./args.sh one two three`
