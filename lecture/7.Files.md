
## 7. Working with Files and Text: The Pipeline Approach

An admin must comfortably combine utilities.

### 7.1. `grep` — pattern search

#### Find all "ERROR" lines in a log, case-insensitive

```bash
grep -i "error" /var/log/syslog
```

#### Recursive search in config files

```bash
grep -r "192.168.1.1" /etc/
```

### 7.2. `awk` — processing and analysis

#### Print 1st and 5th columns

```bash
df -h | awk '{print $1, $5}'
```

#### Sum the size of all .log files

```bash
ls -l *.log | awk '{sum+=$5} END {print "Total size:", sum}'
```

#### Filter by condition: 5th column > 80%

```bash
df -h | awk '$5+0 > 80 {print $1 " is " $5 " full"}'
```

### 7.3. `sed` — stream editor

#### Create test file
```bash
cat <<EOF > file.txt
line 1 old old
line 2 old
line 3 old
line 4 old
# comment 1
EOF
```

#### Replace first occurrence

```bash
sed 's/old/new/' file.txt
```

#### Global replacement (per line)

```bash
sed 's/old/new/g' file.txt
```

#### Delete lines matching a pattern

```bash
sed '/^#/d' file.txt   # remove comments
```

#### In-place edit (with backup)
```bash
sed -i.bak 's/line 2 old/line 22 old/' file.txt
```

### 7.4. `cut` — column selection

```bash
# Extract field 2 with delimiter ':'
cut -d: -f2 /etc/passwd
```

### 7.5. `find` + `xargs` — bulk operations

#### Find and delete files older than 30 days

```bash
find . -name "*.txt" -mmin +1 #-print0
```

- `-print0` is needed to disable newlines in the output

```bash
find . -name "*.txt" -mmin +1 -print0 | xargs -0 rm -f
```
