
## 3. Input, Output, Redirection

### 3.1. Reading User Input
```bash
echo "Enter server name:"
read server_name
echo "Proceeding with configuration for ${server_name}"
```

The `-p` flag provides a prompt without a separate `echo`:

```bash
read -p "Enter port: " port
echo "listen ${port} port"
```

### 3.2. Stream Redirection
- `>` — overwrite a file (stdout).
- `>>` — append to a file.
- `2>` — redirect stderr.
- `2>&1` — merge stderr into stdout.
- `/dev/null` — the "black hole".

```bash
cat <<EOF > ./check_service.sh
echo "msg to stdout"
echo "Fatal: msg to stderr" >&2
exit 1
EOF
chmod +x check_service.sh
```

```bash
# Save command output to a log file, send errors there too
./check_service.sh >> ./admin.log 2>&1
cat ./admin.log
```

### 3.3. Here-Document for Multi-line Input

```bash
cat <<EOF > ./config.conf
server=$(cat /etc/hostname)
port=8080
admin=$(whoami)
EOF
```

Inside, variables are expanded. To prevent substitution, quote the delimiter: `'EOF'`.

### 3.4. Pipes 

- `|` - stdout command_1 to stdin command_2
- `|&` merged stderr and stdout command_1 to stdin command_2

```bash
cat /etc/os-release | grep VERSION
```

```bash
cat /root/.secrets | xargs echo > stdout.log
```

```bash
cat /root/.secrets |& xargs echo > all.log
```
