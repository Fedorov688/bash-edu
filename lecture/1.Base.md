# Bash Scripting for System Administrators: From Zero to Automation

This guide will help a system administrator Bash scripting from the basics up to a level sufficient for everyday automation tasks. We'll start with simple examples and gradually move to real-world administration scenarios.

## 1. What Is Bash and Why Scripts Are Needed

Bash (Bourne Again SHell) is a command interpreter and scripting language available on virtually any Linux/Unix system. A script is simply a text file containing a sequence of commands. Why automate?

- Eliminate routine: backups, log checks, mass user creation
- Increase reliability: avoid typos from manual input
- Document processes: the script itself serves as documentation

**Simplest script example** (file `hello.sh`):
```bash
#!/bin/bash
echo "Hello, ${USER}!"
```

Explanation:
- `#!/bin/bash` — the **shebang**, tells the system which interpreter to use.
- `echo` — prints text to the terminal.

Run it:
```bash
chmod +x hello.sh   # grant execute permission
./hello.sh          # execute
```
