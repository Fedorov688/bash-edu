
## 4. Conditional Statements

### 4.1. `if` Construct
```bash
if [[ -f /etc/passwd ]]; then
    echo "Password file exists"
else
    echo "File not found!"
fi
```

### 4.2. Tests with `test` and `[ ]`
Classic syntax: `test condition` or `[ condition ]` (spaces inside brackets are mandatory). The extended `[[ ... ]]` is recommended-it supports regex, and no word splitting issues.

Common file conditions:
- `-f` - regular file exists.
- `-d` - directory exists.
- `-x` - file is executable.
- `-s` - file size > 0.
- `-nt`/`-ot` - newer/older than (modification time).

String conditions:
- `=` or `==` - equality (with `[[ ]]`, `==` supports patterns).
- `!=` - inequality.
- `-z` - string is empty.
- `-n` - string is not empty.

Numeric conditions:
- `-eq` - equal to `True` if two numbers are equal
- `-ne` - not equal to `True` if the numbers differ
- `-lt` - less than `True` if the first number is smaller than the second
- `-le` - less than or equal to `True` if the first number is smaller or equal
- `-gt` - greater than `True` if the first number is larger than the second
- `-ge` - greater than or equal to `True` if the first number is larger or equal

```bash
age=25
if [[ $age -gt 18 ]]; then
    echo "Adult"
fi
```

### 4.3. Multiple Choice with `case`
```bash
read -p "Enter action (start|stop|restart): " action
case $action in
    start)
        echo "Starting service..."
        ;;
    stop)
        echo "Stopping..."
        ;;
    restart)
        echo "Restarting..."
        ;;
    *)
        echo "Unknown action"
        exit 1
        ;;
esac
```
